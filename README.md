### autodata-tabular

Autodata-tabular is an adapted version of [Autodata](https://arxiv.org/abs/2606.25996), specifically for generating high-quality synthetic tabular data.

Given an initial (grounding) tabular dataset, autodata-tabular uses AI agents equipped with statistical and deep generative modeling tools to iteratively generate, validate, and refine high-fidelity synthetic records:

-   **Challenger:** Analyzes grounding dataset examples to engineer diverse natural language scenarios.
    
-   **Retriever:** Leverages the generated scenario to extract structurally similar records from the baseline pool.
    
-   **Synthetic Generator:** Combines the target scenario and retrieved data points to synthesize a new candidate record.
    
-   **Neural Network Validator:** Maps hidden relationships between baseline features to verify that the generated data obeys complex domain constraints.
    
-   **Distribution Validator:** Benchmarks the new record against historical parameters to confirm its univariate statistical feasibility.
    
-   **Isolation Forest Validator:** Uses an unsupervised anomaly ensemble to ensure the synthetic data point fits cleanly into high-dimensional clusters.
    
-   **Judge:** Orchestrates the validation loop by treating the components above as tools to approve or reject records and deliver iterative feedback.

Traditional tabular synthesis techniques operate strictly within geometric space—drawing arbitrary lines between existing points and injecting randomized noise to fill the gaps. While mathematically rigid, these methods offer zero control over _what_ actually gets generated.

#### Key Advantages
-   **Preserves Deep Data Relationships:** Backed by a deep neural manifold and a multivariate isolation ensemble, the system automatically captures and respects complex, non-linear relationships, mathematical correlations, and raw distributions present in your baseline dataset.
    
-   **Automates High-Dimensional Scaling:** Manually proposing and validating synthetic data rows across dozens of interconnected variables is incredibly cumbersome and prone to human error. Autodata-Tabular completely automates this workflow behind the scenes.
    
-   **Enables Intentional Generation:** Instead of generating randomized data blends, you can use natural language to target specific physical or business scenarios.

**Traditional Synthesis:** _"Generate 500 randomized points between Cluster A and Cluster B."_ (Result: Generic statistical padding)

**Autodata-tabular:** _"Synthesize a player hitting a Driver with an aggressive 180 mph club speed, a slightly open face angle, and a mild fade path into a crosswind."_ (Result: Intentional, domain-compliant, highly specific edge cases.)

### Foundational Framework

Autodata-Tabular is an adapted tabular implementation inspired by the foundational research paper:

> **[Autodata: An agentic data scientist to create high quality synthetic data](https://arxiv.org/abs/2606.25996)** _Published by Meta Fundamental AI Research (FAIR), 2026._ _Authors: Ilia Kulikov, Chenxi Whitehouse, Tianhao Wu, Yixin Nie, Swarnadeep Saha, Eryk Helenowski, Weizhe Yuan, Olga Golovneva, Jack Lanchantin, Yoram Bachrach, Jakob Foerster, Xian Li, Han Fang, Sainbayar Sukhbaatar, Jason Weston._

### Grounding Dataset Source

The core tracking telemetry, physical variables, and baseline dataset used to seed the vector memory and train the deep validation manifolds were sourced from:

> **[Launch Monitor Regression Dataset](https://github.com/tim-blackmore/launch-monitor-regression/tree/main)** _Maintained by Tim Blackmore._ _Provides comprehensive real-world launch monitor profiles (Driver golf data tracking parameters) ideal for testing high-dimensional regression and physical constraint validation loops._
